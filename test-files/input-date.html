input type=date element test

Pass Criteria

  • #1: Element is supported
  • #2: Correct HTML and Core Accessibility API mappings
  • #3: Correct Accessible Name & Description Calculation
  • #4: Control is keyboard accessible:
    • #4.1: Control MUST be focusable
    • #4.2: MUST be able to open date control and select a date

  • HTML5 input type=date element
  • HTML Accessibility API Mappings for input type=date element
