menu type="context" element

Pass Criteria

  • #1: Element is supported
  • #2: Element has correct HTML and Core Accessibility API mappings
  • #3: Control is keyboard accessible

Reference

  • HTML5.1 menuitem element
  • HTML Accessibility API Mappings for menuitem element in the command state
  • Core Accessibility API Mappings for menuitem role
  • HTML Accessibility API Mappings for menuitem element in the checkbox state
  • Core Accessibility API Mappings for menuitemcheckbox role
  • Core Accessibility API Mappings for aria-checked="true" state
  • Core Accessibility API Mappings for aria-checked="false" state
  • HTML Accessibility API Mappings for menuitem element in the radio state
  • Core Accessibility API Mappings for menuitemradio role

Code

<button menu="command-menu">I have a number of command menuitems in my context menu. Right click (or equivilent) to test</button>
<menu type="context" id="command-menu">
	<menuitem type="command" label="I’m a command menuitem" />
	<menuitem type="command" label="I’m another command menuitem" />
</menu>
<button menu="checkbox-menu">I have a number of checkbox menuitems in my context menu. Right click (or equivilent) to test</button>
<menu type="context" id="checkbox-menu">
	<menuitem type="checkbox" label="I’m a checkbox menuitem" checked />
	<menuitem type="checkbox" label="I’m another checkbox menuitem" />
</menu>
<button menu="radio-menu">I have a number of radio menuitems in my context menu. Right click (or equivilent) to test</button>
<menu type="context" id="radio-menu">
	<menuitem type="radio" radiogroup="radios" label="I’m a radio menuitem" checked />
	<menuitem type="radio" radiogroup="radios" label="I’m another radio menuitem" />
</menu>
